## Overview

The Context API provides a way to share values between components without explicitly passing props through every level of the tree. RyunixJS's context implementation uses fiber tree traversal to find the nearest Provider.

## createContext

Creates a context object with a Provider component and a useContext hook.

### Signature

```typescript
function createContext<T>(
  contextId?: string,
  defaultValue?: T
): {
  Provider: ({ value, children }) => Element,
  useContext: (ctxID?: string) => T
}
```

<ParamField path="contextId" type="string">
  Optional unique identifier for the context. Defaults to `RYUNIX_TYPES.RYUNIX_CONTEXT`. Use custom IDs when you need multiple contexts.
</ParamField>

<ParamField path="defaultValue" type="T">
  The default value returned by `useContext` when no Provider is found in the component tree. Defaults to `{}`.
</ParamField>

<ResponseField name="Provider" type="Component">
  A component that accepts `value` and `children` props. Makes the value available to all descendants.
</ResponseField>

<ResponseField name="useContext" type="Hook">
  A hook that retrieves the nearest context value from the component tree.
</ResponseField>

## Usage

### Basic Context

```javascript
import { createContext, useStore } from '@ryunixjs/core'

// Create context
const ThemeContext = createContext('theme-context', { theme: 'light' })

// Root component provides the context
const App = () => {
  const [theme, setTheme] = useStore('light')

  return (
    <ThemeContext.Provider value={{ theme, setTheme }}>
      <Header />
      <MainContent />
      <Footer />
    </ThemeContext.Provider>
  )
}

// Nested components consume the context
const Header = () => {
  const { theme, setTheme } = ThemeContext.useContext()

  return (
    <header className={theme}>
      <button onClick={() => setTheme(theme === 'light' ? 'dark' : 'light')}>
        Toggle Theme
      </button>
    </header>
  )
}

const MainContent = () => {
  const { theme } = ThemeContext.useContext()
  return <main className={theme}>Content here</main>
}
```

### Multiple Contexts

```javascript
const UserContext = createContext('user-context', null)
const SettingsContext = createContext('settings-context', {})

const App = () => {
  const [user, setUser] = useStore(null)
  const [settings, setSettings] = useStore({ notifications: true })

  return (
    <UserContext.Provider value={{ user, setUser }}>
      <SettingsContext.Provider value={{ settings, setSettings }}>
        <Dashboard />
      </SettingsContext.Provider>
    </UserContext.Provider>
  )
}

const Dashboard = () => {
  const { user } = UserContext.useContext()
  const { settings } = SettingsContext.useContext()

  return (
    <div>
      <div>Welcome, {user?.name}</div>
      <div>Notifications: {settings.notifications ? 'On' : 'Off'}</div>
    </div>
  )
}
```

### Context with Reducer

```javascript
import { createContext, useReducer } from '@ryunixjs/core'

const CartContext = createContext('cart-context', { items: [], total: 0 })

const cartReducer = (state, action) => {
  switch (action.type) {
    case 'add':
      const newItems = [...state.items, action.item]
      return {
        items: newItems,
        total: newItems.reduce((sum, item) => sum + item.price, 0)
      }
    case 'remove':
      const filtered = state.items.filter(item => item.id !== action.id)
      return {
        items: filtered,
        total: filtered.reduce((sum, item) => sum + item.price, 0)
      }
    case 'clear':
      return { items: [], total: 0 }
    default:
      return state
  }
}

const CartProvider = ({ children }) => {
  const [state, dispatch] = useReducer(cartReducer, { items: [], total: 0 })

  return (
    <CartContext.Provider value={{ state, dispatch }}>
      {children}
    </CartContext.Provider>
  )
}

const ProductCard = ({ product }) => {
  const { dispatch } = CartContext.useContext()

  return (
    <div>
      <h3>{product.name}</h3>
      <p>${product.price}</p>
      <button onClick={() => dispatch({ type: 'add', item: product })}>
        Add to Cart
      </button>
    </div>
  )
}

const Cart = () => {
  const { state, dispatch } = CartContext.useContext()

  return (
    <div>
      <h2>Cart ({state.items.length} items)</h2>
      <div>Total: ${state.total}</div>
      {state.items.map(item => (
        <div key={item.id}>
          {item.name} - ${item.price}
          <button onClick={() => dispatch({ type: 'remove', id: item.id })}>
            Remove
          </button>
        </div>
      ))}
      <button onClick={() => dispatch({ type: 'clear' })}>Clear Cart</button>
    </div>
  )
}
```

### Custom Hook Pattern

```javascript
import { createContext, useStore } from '@ryunixjs/core'

const AuthContext = createContext('auth-context', null)

// Custom hook for better ergonomics
const useAuth = () => {
  const context = AuthContext.useContext()
  if (!context) {
    throw new Error('useAuth must be used within AuthProvider')
  }
  return context
}

const AuthProvider = ({ children }) => {
  const [user, setUser] = useStore(null)
  const [loading, setLoading] = useStore(true)

  const login = async (credentials) => {
    setLoading(true)
    try {
      const response = await fetch('/api/login', {
        method: 'POST',
        body: JSON.stringify(credentials)
      })
      const userData = await response.json()
      setUser(userData)
    } finally {
      setLoading(false)
    }
  }

  const logout = () => {
    setUser(null)
  }

  return (
    <AuthContext.Provider value={{ user, loading, login, logout }}>
      {children}
    </AuthContext.Provider>
  )
}

const LoginButton = () => {
  const { user, login, logout } = useAuth()

  if (user) {
    return <button onClick={logout}>Logout</button>
  }

  return (
    <button onClick={() => login({ email: 'user@example.com', password: 'pass' })}>
      Login
    </button>
  )
}
```

### Nested Providers (Same Context)

```javascript
const LanguageContext = createContext('language-context', 'en')

const App = () => {
  return (
    <LanguageContext.Provider value="en">
      <div>Default: English</div>
      <LanguageContext.Provider value="es">
        <div>Spanish Section</div>
        <NestedComponent />
      </LanguageContext.Provider>
    </LanguageContext.Provider>
  )
}

const NestedComponent = () => {
  // Gets "es" from the nearest Provider
  const language = LanguageContext.useContext()
  return <div>Language: {language}</div>
}
```

## Server-Side Rendering

During SSR, contexts are stored in `state.ssrContexts`:

```javascript
const MyContext = createContext('my-context', { value: 'default' })

// During SSR, the context value is stored and retrieved from ssrContexts
const Component = () => {
  const value = MyContext.useContext()
  // value comes from state.ssrContexts['my-context'] or defaultValue
}
```

## Implementation Details

### Provider Component

The Provider creates a special fiber node with `_contextId` and `_contextValue` properties:

```javascript
const Provider = ({ value, children }) => {
  return createElement(
    RYUNIX_TYPES.RYUNIX_CONTEXT,
    { value, children, _contextId: contextId },
    ...flattenArray([children])
  )
}
```

### useContext Traversal

`useContext` walks up the fiber tree to find the nearest Provider:

```javascript
let fiber = state.wipFiber

while (fiber) {
  if (fiber._contextId === ctxID && fiber._contextValue !== undefined) {
    return fiber._contextValue
  }
  if (fiber.type?._contextId === ctxID && fiber.props?.value !== undefined) {
    return fiber.props.value
  }
  fiber = fiber.parent
}
return defaultValue
```

## Best Practices

### When to Use Context

- Global app state (theme, auth, language)
- Deeply nested props (avoiding prop drilling)
- Shared utilities or services
- Configuration that many components need

### When NOT to Use Context

- Frequently changing values (can cause many re-renders)
- Performance-critical paths (prefer local state)
- Simple parent-child communication (use props)

### Performance Optimization

```javascript
import { useMemo } from '@ryunixjs/core'

const ExpensiveProvider = ({ children }) => {
  const [state, dispatch] = useReducer(reducer, initialState)

  // Memoize context value to prevent unnecessary re-renders
  const value = useMemo(() => ({ state, dispatch }), [state])

  return <Context.Provider value={value}>{children}</Context.Provider>
}
```

### Split Contexts

```javascript
// ❌ Bad - one large context causes all consumers to re-render
const AppContext = createContext('app', {
  user: null,
  theme: 'light',
  settings: {},
  notifications: []
})

// ✅ Good - separate contexts allow fine-grained subscriptions
const UserContext = createContext('user', null)
const ThemeContext = createContext('theme', 'light')
const SettingsContext = createContext('settings', {})
const NotificationsContext = createContext('notifications', [])
```

## Behavior

- Providers can be nested; `useContext` returns the value from the nearest ancestor
- Context values can be any JavaScript value (primitives, objects, functions)
- During SSR, contexts are stored in `state.ssrContexts`
- Validates hook context in development mode
- Returns `defaultValue` if no Provider is found
